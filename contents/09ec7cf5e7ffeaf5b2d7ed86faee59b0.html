Modificar Ordem de Compra

UMLUseCase
Untitled :: Model :: Modificar Ordem de Compra

Description

Descrição: Este caso de uso descreve o processo pelo qual um usuário pode modificar uma Ordem de Compra (OC) previamente enviada por meio do sistema Home Broker da Soy Stocks (S2). A funcionalidade permite que os usuários ajustem os detalhes de uma OC antes de sua execução, oferecendo flexibilidade na gestão de transações.

Ator Principal: Usuário

Stakeholders e Interesses: Usuário: Interessado em ajustar os detalhes de uma OC que foi previamente enviada, mas ainda não foi executada. Soy Stocks (S2): Interessada em fornecer uma interface eficiente e segura para os usuários gerenciarem suas ordens de compra.

Pré-condições: O usuário deve estar autenticado no sistema. Deve existir uma OC previamente enviada pelo usuário que ainda não foi executada.

Fluxo Básico: O usuário acessa a seção de ordens no Home Broker. O usuário visualiza a lista de OCs enviadas e identifica a que deseja modificar. O usuário seleciona a opção "Modificar" associada à OC desejada. O sistema exibe os detalhes atuais da OC, permitindo que o usuário faça as modificações desejadas, como ajuste da quantidade de ações, preço ou outros parâmetros. O usuário confirma as modificações. O sistema valida as modificações, verificando se ainda são permitidas (por exemplo, se a OC ainda não foi executada). Se as modificações são válidas, o sistema atualiza os detalhes da OC conforme as modificações fornecidas pelo usuário. O usuário recebe uma confirmação das modificações da OC.

Fluxos Alternativos: Se a OC já foi executada, o sistema informa ao usuário que não é possível modificá-la. Se houver problemas técnicos durante o processo de modificação, o sistema deve informar ao usuário sobre a impossibilidade temporária de realizar essa operação.

Pós-condições: Os detalhes da OC são atualizados com as modificações fornecidas pelo usuário.

Regras de Negócio: O sistema deve permitir a modificação apenas de OCs que ainda não foram executadas. O sistema pode impor limites nas modificações permitidas, como restrições na alteração do preço da ação.

Exceções: Se houver problemas técnicos durante o processo de modificação, o sistema deve informar ao usuário sobre a impossibilidade temporária de realizar essa operação.

Notas Adicionais: Essa funcionalidade oferece aos usuários a capacidade de ajustar detalhes de suas OCs, permitindo uma maior personalização e adaptabilidade em suas estratégias de investimento. É crucial garantir que as modificações sejam permitidas apenas para OCs que ainda não foram executadas.
