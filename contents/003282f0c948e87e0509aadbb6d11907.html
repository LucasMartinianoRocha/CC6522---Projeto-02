Enviar OV por Quantidade de Ações

UMLUseCase
Untitled :: Model :: Enviar OV por Quantidade de Ações

Description

Descrição: Este caso de uso descreve o processo pelo qual um usuário, por meio da interface do sistema Home Broker da Soy Stocks (S2), envia uma ordem de venda de ações para a Bolsa de Valores de São Paulo (B3), especificando a quantidade de ações a serem vendidas. Essa funcionalidade é essencial para permitir que os usuários controlem o volume de ações a serem transacionadas em uma ordem de venda.

Ator Principal: Usuário

Stakeholders e Interesses: Usuário: Interessado em vender uma quantidade específica de ações por meio do Home Broker. Soy Stocks (S2): Interessada em fornecer uma plataforma eficiente e personalizável para que os usuários realizem transações de venda de ações. Bolsa de Valores de São Paulo (B3): Interessada em receber e processar ordens de venda com especificações claras sobre a quantidade de ações a serem transacionadas.

Pré-condições: O usuário deve estar autenticado no sistema Home Broker. O usuário deve possuir as ações que pretende vender. O sistema Home Broker deve estar devidamente autenticado e autorizado para enviar ordens de venda para a B3.

Fluxo Básico: O usuário acessa a interface do Home Broker e inicia o processo de venda de suas ações, especificando a quantidade desejada. O sistema Home Broker valida a ordem, verificando se todos os parâmetros necessários estão corretos e se o usuário possui as ações que pretende vender. O sistema gera a mensagem padronizada para a ordem de venda, incluindo a quantidade de ações especificada, e a envia para a Bolsa de Valores (B3) por meio de APIs de integração. A Bolsa de Valores recebe a ordem, valida-a e a processa conforme as regras do mercado. A Bolsa de Valores envia uma confirmação de execução ou rejeição da ordem de volta ao sistema Home Broker. O sistema Home Broker atualiza o status da ordem no sistema do usuário, refletindo se a ordem foi executada ou rejeitada.

Fluxos Alternativos: Se a validação da ordem falhar, o sistema Home Broker informa ao usuário sobre os problemas encontrados e não envia a ordem de venda para a B3. Se houver problemas técnicos durante o envio da ordem de venda para a B3, o sistema Home Broker deve informar ao usuário sobre a impossibilidade temporária de realizar essa operação.

Pós-condições: O usuário recebe uma confirmação do sistema Home Broker sobre o status da ordem de venda. O sistema Home Broker atualiza o histórico de transações do usuário.

Regras de Negócio: O sistema Home Broker deve seguir os padrões e protocolos estabelecidos pela Bolsa de Valores para o envio de ordens de venda. A validação da ordem de venda deve ser rigorosa para garantir que apenas ordens válidas sejam enviadas para a B3.

Exceções: Se a validação da ordem de venda falhar, o sistema Home Broker deve informar ao usuário sobre os problemas encontrados e impedir o envio da ordem. Se houver problemas técnicos durante o envio da ordem de venda para a B3, o sistema Home Broker deve informar ao usuário sobre a impossibilidade temporária de realizar essa operação.

Notas Adicionais: Esta funcionalidade oferece flexibilidade ao usuário, permitindo que ele especifique a quantidade exata de ações a serem vendidas em uma ordem, proporcionando maior controle sobre suas transações.
