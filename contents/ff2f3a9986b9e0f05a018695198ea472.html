Logar no Sistema

UMLUseCase
Untitled :: Model :: Logar no Sistema

Description

Descrição: Este caso de uso descreve o processo pelo qual um usuário acessa o sistema da Soy Stocks (S2) através do processo de login. O login no sistema é necessário para que o usuário possa utilizar as funcionalidades do Home Broker, realizar transações, e acessar informações personalizadas.

Ator Principal: Usuário

Stakeholders e Interesses: Usuário: Interessado em acessar o sistema para realizar operações no Home Broker. Soy Stocks (S2): Interessada em garantir a autenticação segura dos usuários e o acesso controlado ao sistema.

Pré-condições: O usuário deve ter uma conta cadastrada no sistema da Soy Stocks (S2). O sistema da Soy Stocks (S2) deve estar operacional.

Fluxo Básico: O usuário acessa a página de login do sistema. O usuário insere seu nome de usuário (ou e-mail) e senha nos campos apropriados. O sistema verifica a validade das credenciais do usuário. Se as credenciais são válidas, o sistema autentica o usuário e concede acesso ao sistema. O usuário é redirecionado para a página inicial ou uma área personalizada após o login bem-sucedido.

Fluxos Alternativos: Se as credenciais fornecidas são inválidas, o sistema notifica o usuário sobre a falha de autenticação e fornece a oportunidade de corrigir as informações. O sistema pode implementar medidas adicionais de segurança, como autenticação de dois fatores, dependendo das configurações do usuário.

Pós-condições: O usuário está autenticado no sistema e tem acesso às funcionalidades autorizadas.

Regras de Negócio: As senhas fornecidas pelo usuário durante o login devem atender a critérios de segurança especificados.

Exceções: Se houver problemas técnicos durante o processo de login, o sistema deve informar o usuário sobre a impossibilidade de acessar temporariamente e sugerir tentar novamente mais tarde.

Notas Adicionais: A segurança durante o processo de login é crítica para garantir a proteção das contas dos usuários e a integridade do sistema. Medidas de segurança, como bloqueio temporário após várias tentativas falhas, podem ser implementadas.
