Enviar OV com Gatilho

UMLUseCase
Untitled :: Model :: Enviar OV com Gatilho

Description

Descrição: Este caso de uso descreve o processo pelo qual um usuário, por meio da interface do sistema Home Broker da Soy Stocks (S2), envia uma ordem de venda de ações para a Bolsa de Valores de São Paulo (B3) com a especificação de um "gatilho". Essa funcionalidade permite que os usuários programem automaticamente uma ordem de venda caso determinadas condições de mercado sejam atendidas.

Ator Principal: Usuário

Stakeholders e Interesses: Usuário: Interessado em programar uma ordem de venda automática com base em condições específicas de mercado. Soy Stocks (S2): Interessada em fornecer uma plataforma eficiente e personalizável para que os usuários realizem transações de venda de ações, incluindo ordens com gatilho. Bolsa de Valores de São Paulo (B3): Interessada em receber e processar ordens de venda com gatilho de maneira eficiente e transparente.

Pré-condições: O usuário deve estar autenticado no sistema Home Broker. O usuário deve possuir as ações que pretende vender. O sistema Home Broker deve estar devidamente autenticado e autorizado para enviar ordens de venda com gatilho para a B3.

Fluxo Básico: O usuário acessa a interface do Home Broker e inicia o processo de configuração de uma ordem de venda com gatilho. O usuário especifica as condições de mercado que acionarão automaticamente a ordem de venda, como um determinado preço de mercado. O sistema Home Broker valida as condições fornecidas pelo usuário. O usuário confirma as configurações da ordem de venda com gatilho. Quando as condições de mercado especificadas são atendidas, o sistema Home Broker gera a mensagem padronizada para a ordem de venda e a envia para a Bolsa de Valores (B3) por meio de APIs de integração. A Bolsa de Valores recebe a ordem, processa-a e a executa automaticamente quando as condições de mercado são satisfeitas. A Bolsa de Valores envia uma confirmação de execução da ordem de volta ao sistema Home Broker. O sistema Home Broker atualiza o status da ordem no sistema do usuário, refletindo que a ordem foi executada automaticamente devido ao gatilho.

Fluxos Alternativos: Se a validação das condições de mercado falhar, o sistema Home Broker informa ao usuário sobre os problemas encontrados e impede a configuração da ordem de venda com gatilho. Se houver problemas técnicos durante o envio da ordem de venda com gatilho para a B3, o sistema Home Broker deve informar ao usuário sobre a impossibilidade temporária de realizar essa operação.

Pós-condições: O usuário recebe uma confirmação do sistema Home Broker sobre o status da ordem de venda com gatilho. O sistema Home Broker atualiza o histórico de transações do usuário.

Regras de Negócio: O sistema Home Broker deve seguir os padrões e protocolos estabelecidos pela Bolsa de Valores para o envio de ordens de venda com gatilho. As condições de mercado configuradas pelo usuário devem ser válidas e relevantes para a execução automática da ordem.

Exceções: Se a validação das condições de mercado falhar, o sistema Home Broker deve informar ao usuário sobre os problemas encontrados e impedir a configuração da ordem de venda com gatilho. Se houver problemas técnicos durante o envio da ordem de venda com gatilho para a B3, o sistema Home Broker deve informar ao usuário sobre a impossibilidade temporária de realizar essa operação.

Notas Adicionais: Essa funcionalidade oferece aos usuários a capacidade de automatizar suas estratégias de venda, permitindo que eles programem ordens de venda com base em condições de mercado específicas, proporcionando maior flexibilidade e controle sobre suas transações.
